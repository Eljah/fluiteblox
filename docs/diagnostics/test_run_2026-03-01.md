## Conclusions

1. The project now builds and runs `mvn test` in this container after downloading/installing required Android SDK artifacts.
2. Current tests are not executed as JUnit tests (`Tests run: 0`) because test classes are utility/main-style and are not discovered as test methods by Surefire.
3. The OpenCV experiment utility still cannot execute in OpenCV mode in this environment (falls back to legacy path), even after JNI package installation.
4. For reliable automated regression of the experiment scenario, it is recommended to convert utility mains into proper JUnit tests and explicitly assert OpenCV initialization diagnostics.
